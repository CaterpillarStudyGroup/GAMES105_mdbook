# 混合方法

> &#x2705; **本章定位**：理解如何**在动力学方法中引入运动学数据**，实现高质量且符合物理的角色动画。

---

## 核心问题

**动力学方法的困境**：
- 纯动力学方法（如轨迹优化、SIMBICON）符合物理、抗扰动
- 但动作质量低（像机器人）、需要大量调参/训练

**关键问题**：如何利用丰富的运动学数据（动捕、动作库）来提升动力学方法的动作质量？

---

## 混合方法的本质

```
┌─────────────────────────────────────────────────────────┐
│  混合方法 = 动力学方法 + 运动学数据                      │
│                                                         │
│  动力学方法提供：物理仿真、抗扰动能力                    │
│  运动学数据提供：高质量动作参考、风格先验                │
└─────────────────────────────────────────────────────────┘
```

**与前面章节的关系**：

```
运动学方法 ──→ 提供数据/先验 ──┐
                              ├──→ 混合方法
动力学方法 ──→ 提供仿真/执行 ──┘
```

---

## 引入运动学数据的方式

### 方式 1：作为跟踪目标

```
动捕轨迹 → 目标 → RL 学习跟踪策略 → 仿真器
              (DeepMimic)
```

**特点**：
- 运动学数据：**完整的动作轨迹**
- 动力学方法：**RL 学习跟踪策略**
- 依赖程度：**强依赖**，推理时仍需参考轨迹

**深入学习**：[DeepMimic](https://caterpillarstudygroup.github.io/ReadPapers/201.html)

---

### 方式 2：作为风格先验

```
动捕数据 → 训练判别器（学习风格） → RL 策略生成动作
              (AMP)
```

**特点**：
- 运动学数据：**风格标签/判别器**
- 动力学方法：**RL 生成动作**
- 依赖程度：**弱依赖**，推理时无需参考轨迹

**深入学习**：[AMP](https://caterpillarstudygroup.github.io/ReadPapers/198.html)

---

### 方式 3：作为初值引导

```
动作数据库 → Motion Matching → 初值 → 轨迹优化 → 仿真器
              (DReCon)
```

**特点**：
- 运动学数据：**动作数据库**
- 动力学方法：**轨迹优化做物理适配**
- 依赖程度：**中等**，推理时需要数据库

**深入学习**：[DReCon](https://caterpillarstudygroup.github.io/ReadPapers/190.html)

---

### 方式 4：作为训练数据

```
动捕数据 → 训练 Diffusion 模型 → 输出力矩/姿态
           (UniPhys/CAMDM)
```

**特点**：
- 运动学数据：**状态 - 动作对**
- 动力学方法：**端到端学习策略**
- 依赖程度：**中等**，推理时需要条件输入

**深入学习**：[UniPhys](https://caterpillarstudygroup.github.io/ReadPapers/184.html) | [CAMDM](https://caterpillarstudygroup.github.io/ReadPapers/177.html)

## 对比表格

| 方法 | 运动学数据形式 | 动力学方法 | 依赖程度 | 推理时需要数据 |
|------|--------------|-----------|---------|--------------|
| **DeepMimic** | 完整轨迹 | RL 跟踪 | 强 | 是（参考轨迹） |
| **AMP** | 风格先验 | RL 生成 | 弱 | 否 |
| **DReCon** | 动作数据库 | 轨迹优化 | 中 | 是（数据库） |
| **UniPhys** | 状态 - 动作对 | 端到端策略 | 中 | 是（条件输入） |

## 本章重点问题

1. **如何设计运动学数据与动力学方法的接口？**
   - DeepMimic：参考轨迹作为 reward
   - AMP：动捕数据训练判别器
   - DReCon：数据库提供初值

2. **如何平衡数据依赖与泛化能力？**
   - 强依赖：动作质量高，但泛化差
   - 弱依赖：泛化好，但训练难

3. **如何处理训练与推理的差异？**
   - 训练时：可以用动捕数据
   - 推理时：根据方法不同，依赖程度不同
