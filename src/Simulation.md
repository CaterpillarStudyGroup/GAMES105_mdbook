## 仿真器能做什么？

在基于动力学的角色动画中，我们需要两个核心组件：

1. **仿真器 (Simulator)**：模拟物理世界，计算角色在力和约束作用下的运动
2. **控制器 (Controller)**：生成关节力矩，驱动角色完成目标动作

本章关注**仿真器**部分，控制器部分将在后续章节（PD 控制、轨迹优化、强化学习）详细介绍。

## 角色仿真基础

### 刚体与关节

角色的物理简化模型由以下元素组成：

**刚体 (Rigid Bodies)**：
- 质量 \\(m_i\\)、转动惯量 \\(I_i\\)
- 位置 \\(x_i\\)、旋转 \\(R_i\\)
- 碰撞几何体（盒体、球体、胶囊体等）

**关节 (Joints)**：
- 位置
- 类型（铰链、球窝等，决定约束方程）
- 连接的刚体

> &#x2705; 关节的数量比刚体的数量少 1

![](./assets/09-06.png)

> &#x2705; 仿真过程中通常使用简单几何体代替 Mesh，便于碰撞检测计算。

## 仿真 Pipeline

### 无接触情况的仿真

```
当前状态 → 计算外力 → 积分 → 下一帧状态
```

### 有接触的仿真

```
当前状态 → 计算外力 → 约束求解 → 积分 → 下一帧状态
              ↓
         接触检测与响应
```

### Ragdoll 仿真流程

![](./assets/08-18.png)

1. **黄色**：计算当前状态
2. **绿色**：约束求解，解出下一时刻的速度
3. **蓝色**：积分更新位置

> &#x2705; 这是最基础的「布娃娃」效果，缺少主动力 \\(f\\) 驱动角色产生有目的的运动。

## 仿真器的局限性

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 穿模 | 离散时间步长、Penalty 方法参数不当 | 减小步长、约束建模 |
| 数值不稳定 | 刚性系统、高增益 PD 控制 | 隐式积分、阻尼 |
| 计算成本高 | 多刚体系统、复杂接触 | 简化模型、并行计算 |

---

## 从仿真到控制

仿真器本身只能被动响应外力和约束。要让角色主动运动，需要**控制器**生成关节力矩：

```
控制器 → 关节力矩 → 仿真器 → 角色运动
         ↑                      ↓
         └────── 状态反馈 ──────┘
```

控制器的设计方法包括：
- **PD 控制**：跟踪目标轨迹
- **轨迹优化**：求解最优控制序列
- **强化学习**：学习端到端控制策略

这些内容将在后续章节详细讨论。
